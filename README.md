# Kubernetes through Incidents and Stories

A book about Kubernetes concepts explained through real incidents and stories.

## Repository Structure

```
k8sbook/
├── book.toml           # mdBook configuration
├── src/                # Book source files (edit these)
│   ├── SUMMARY.md      # Table of contents — controls chapter order
│   ├── cover.md        # Cover page (displays cover.png)
│   ├── cover.png       # Cover image
│   ├── Introduction.md # Introduction (not in book currently — add to SUMMARY.md to include)
│   ├── *.md            # Chapter source files
│   └── *.png / *.yaml  # Images and supporting files
├── theme/
│   └── custom.css      # Custom styles (cover page, print/PDF layout)
└── book/               # Build output — generated, not committed
    ├── html/           # HTML book
    └── pdf/            # PDF export
```

## Prerequisites

```bash
# Install mdbook
brew install mdbook

# Install PDF backend (one-time, takes ~2 minutes)
cargo install mdbook-pdf

# Add cargo to PATH (add this line to ~/.zshrc to make it permanent)
export PATH="$PATH:/Users/senthil/.cargo/bin"
```

## Building the Book

```bash
# Build both HTML and PDF
export PATH="$PATH:/Users/senthil/.cargo/bin"
mdbook build

# Open the HTML book in browser
open book/html/index.html

# Open the PDF
open book/pdf/output.pdf

# Live-reload server while writing (HTML only)
mdbook serve --open
```

## Workflow — Adding or Editing Content

1. **Edit a chapter** — open the `.md` file in `src/` and edit it directly.

2. **Add a new chapter:**
   - Create a new `.md` file in `src/`
   - Add an entry to `src/SUMMARY.md` in the `# Chapters` section:
     ```markdown
     - [Chapter N - Your Title](Your%20File%20Name.md)
     ```
   - Run `mdbook build`

3. **Change chapter order** — reorder the entries in `src/SUMMARY.md`.

4. **Add images** — place image files in `src/` and reference them in markdown:
   ```markdown
   ![alt text](image-name.png)
   ```

5. **Rebuild after any change:**
   ```bash
   export PATH="$PATH:/Users/senthil/.cargo/bin"
   mdbook build
   ```

## Notes

- `book/` is git-ignored — it is always regenerated by `mdbook build`.
- `Introduction.md` is kept as a draft. To add it back to the book, insert `[Introduction](Introduction.md)` before the `# Chapters` line in `SUMMARY.md`.
- Each chapter automatically starts on a new page in the PDF.
